คำถามที่พบบ่อยเกี่ยวกับสกิล `/dubbing` และ `/srt` สำหรับการตั้งค่าและวิธีใช้งาน โปรดดูที่ [README](README.md)

### ต้องมีอะไรบ้างในการใช้งาน?

Node.js 18+ และคีย์ API ของ Perso Dubbing ติดตั้งสกิลแล้วพูดว่า *"พากย์วิดีโอนี้ให้หน่อย"* → [รับคีย์ API](https://developers.perso.ai/api-keys)

### จะลงทะเบียนคีย์ API ของฉันได้อย่างไร?

ในการรันครั้งแรกจะมีไฟล์คีย์เปิดขึ้นมาโดยอัตโนมัติ — วาง**เฉพาะคีย์ API ของคุณ**ลงในไฟล์นั้นแล้วบันทึก (คีย์จะถูกเข้ารหัสและไฟล์จะถูกลบทิ้ง) **ห้ามวางคีย์ลงในแชทเด็ดขาด** ตรวจสอบด้วยตนเอง: `npm run key:check`

### มีค่าใช้จ่ายหรือไม่?

โค้ดของสกิลนี้ฟรี (MIT) แต่การพากย์เสียงทำงานผ่าน Perso API ซึ่งคิดค่าใช้จ่ายเป็นเครดิต: การพากย์เสียง ≈ 1 เครดิต/วินาที ลิปซิงก์ ≈ ×2 การแยกเสียง ≈ ×0.5 ไฟล์ต้นฉบับความละเอียด 4K จะถูกคิดค่าใช้จ่าย ×3 ในแผน pro/business/enterprise การคิดเครดิตจากฝั่งเซิร์ฟเวอร์ถือเป็นค่าที่ถูกต้องที่สุด

### ใช้ไฟล์แบบไหนได้บ้าง?

ไฟล์ในเครื่อง ทั้งโฟลเดอร์ (แบบชุด) หรือ URL — รวมถึง YouTube, TikTok, Google Drive และ Vimeo วิดีโอที่มีขนาดใหญ่เกินไปหรือยาวมากจะถูกแบ่ง ประมวลผล และรวมกลับเข้าด้วยกันโดยอัตโนมัติ

### พากย์เสียงหลายภาษา หรือประมวลผลหลายไฟล์พร้อมกันได้ไหม?

ได้ ใส่หลายภาษาไว้ในคำสั่งเดียว (`--target en,ja,zh`) — ต้นฉบับจะถูกอัปโหลดและแบ่งเพียงครั้งเดียว แล้วนำกลับมาใช้ซ้ำในแต่ละภาษา คุณยังสามารถผสมไฟล์ โฟลเดอร์ และ URL หลายรายการในการรันครั้งเดียวได้อีกด้วย

### ผลลัพธ์ถูกบันทึกไว้ที่ไหน?

ตามค่าเริ่มต้นจะบันทึกไว้ข้างวิดีโอต้นฉบับ หรือในโฟลเดอร์ที่คุณระบุด้วย `--out` การรันแต่ละครั้งยังเป็นโปรเจกต์ในพอร์ทัล Perso ของคุณด้วย (<https://portal.perso.ai>) ซึ่งคุณสามารถดาวน์โหลดซ้ำหรือขอรูปแบบไฟล์อื่นได้ที่นั่น

### ลิปซิงก์คืออะไร?

เป็นการปรับการขยับปากให้ตรงกับเสียงที่พากย์ไว้ ทำงานหลังจากการพากย์เสียง ใช้ได้กับวิดีโอเท่านั้น ใช้เวลานานกว่ามาก และเสียเครดิตเพิ่มเติม เพิ่ม `--lipsync`

### การแยกเสียงคืออะไร?

เป็นการแบ่งต้นฉบับออกเป็นแทร็กเสียงพูด / เสียงพื้นหลัง / เสียงพื้นหลังรอง — ไม่มีการพากย์เสียงเกี่ยวข้อง เพิ่ม `--separate`

### ทำคำบรรยาย (SRT) แทนการพากย์เสียงได้ไหม?

ได้ — แพ็กเกจนี้ยังติดตั้งสกิล **`/srt`** ให้ด้วย โดยจะดึงคำบรรยายภาษาต้นฉบับออกจากวิดีโอ/เสียง/URL ผ่านระบบแปลงเสียงพูดเป็นข้อความของ Perso จากนั้นเอเจนต์ของคุณจะแปลเป็นภาษาที่คุณต้องการ (บันทึกเป็น `<name>_<lang>_Subtitle.srt` ไว้ข้างไฟล์ต้นฉบับ) ต้องการแค่บทถอดเสียงใช่ไหม? บอกไว้แล้วรันด้วย `--transcribe-only` — ไม่มีการแปล การดึงคำบรรยายแต่ละครั้งจะใช้เครดิตตามสัดส่วนความยาวของสื่อ (ต่อภาษา)

### หยุดทำงานกลางคัน (เครดิตหมด ขัดข้อง หรือเชลล์ถูกยกเลิก) ต้องทำอย่างไร?

ความคืบหน้าจะถูกบันทึกลงในไฟล์สถานะตลอดการทำงาน (`*.dubresume.json` สำหรับ `/dubbing`, `*.srtresume.json` สำหรับ `/srt`) รันคำสั่ง `--resume "<state-file>"` ที่แสดงไว้อีกครั้งเพื่อทำเฉพาะส่วนที่เหลือให้เสร็จ — ส่วนที่เสร็จแล้วจะถูกข้ามและจะไม่ถูกคิดเครดิตซ้ำ

### เครดิตหมด จะเติมเครดิตได้อย่างไร?

สกิลสามารถสร้างลิงก์ชำระเงินผ่าน Stripe ได้ (สมัครสมาชิก เปลี่ยนแผน หรือซื้อเครดิต ขึ้นอยู่กับแผนของคุณ) คุณจะเป็นผู้เปิดลิงก์และชำระเงินด้วยตัวเอง — เอเจนต์จะไม่ชำระเงินแทนคุณ หลังจากเติมเครดิตแล้ว ให้ทำงานต่อด้วยคำสั่ง `--resume` ที่แสดงไว้

### พากย์เสียงโดยไม่บันทึกไฟล์ในเครื่องได้ไหม?

ได้ สำหรับวิดีโอเดี่ยว (ที่ไม่ได้ถูกแบ่ง): เพิ่ม `--no-save` ผลลัพธ์จะยังคงอยู่ใน workspace ของ Perso และจะไม่ถูกดาวน์โหลด ส่วนวิดีโอที่ถูกแบ่งจะยังคงถูกบันทึกตามปกติ เนื่องจากไฟล์ที่รวมแล้วจำเป็นต้องดาวน์โหลดมาไว้ในเครื่อง

### ไม่พบ `node` — ต้องทำอย่างไร?

สกิลนี้ต้องการ Node.js 18+ ตรวจสอบด้วย `node -v` ติดตั้งเวอร์ชัน LTS จาก <https://nodejs.org> หรือขอให้เอเจนต์ติดตั้งให้คุณ แล้วลองใหม่อีกครั้ง

### จะอัปเดตสกิลได้อย่างไร?

`npx perso-dubbing@latest` หรือในปลั๊กอิน Claude Code: `/plugin update perso-dubbing`

### สกิลเก็บข้อมูลอะไรบ้าง?

เก็บเฉพาะข้อมูลการใช้งานแบบไม่ระบุตัวตนเท่านั้น — การทำงานที่รัน ว่าสำเร็จหรือไม่ ตัวนับแบบคร่าวๆ เวอร์ชันแอป และระบบปฏิบัติการ — ติดแท็กด้วย ID แบบสุ่มต่อการติดตั้ง ไม่มีการรวมคีย์ API ชื่อไฟล์หรือเนื้อหาสื่อ บัญชี/อีเมล หรือ workspace ID ของคุณแต่อย่างใด
