Навык (skill) для агентов программирования, который добавляет вашему агенту ИИ-дубляж от [Perso Dubbing](https://perso.ai/dubbing). Установите его один раз, а затем просто скажите *«продублируй это видео на английский»*.

- **Дубляж** на другой язык — один файл, целая папка или URL
- **Синхронизация губ** в дублированном видео, чтобы движения рта совпадали с новой озвучкой
- **Разделение** голоса и фонового звука
- **Субтитры** (`/srt`) — извлечение SRT через распознавание речи, после чего ваш агент их переводит
- Слишком большие и очень длинные материалы автоматически разбиваются на части, обрабатываются и снова объединяются

Работает на **Node.js 18+** и требует **API-ключ Perso Dubbing**. Построен на стандарте Agent Skills (`SKILL.md`), поэтому ведёт себя одинаково в Claude, Codex, Cursor и Antigravity.

## Установка

> 📖 **[Наглядная инструкция →](https://dubbing-plugin.perso.ai/en/)** — выберите агента и скопируйте команды.

### Приложение Claude для компьютера — проще всего, без терминала (около 3 минут)

В <a href="https://claude.ai/download" target="_blank" rel="noopener noreferrer">приложении Claude для компьютера</a> (платный план):

1. Откройте **вкладку Code** (по центру сверху), выберите любую папку и укажите окружение **Local** — в облачных сессиях плагины не работают.
2. Вставляйте команды в поле ввода по одной и нажимайте Enter:

   ```text
   claude marketplace add perso-ai/perso-dubbing-plugin
   ```

   ```text
   claude install perso-dubbing@perso-ai
   ```

   *Предпочитаете клики? После первой команды: кнопка **+** → **Plugins** → **Add plugin** → **perso-dubbing**.*
3. Попросите продублировать видео — *«Продублируй это видео на английский — C:\videos\clip.mp4»*. URL с YouTube или папка тоже подойдут. Если не указать `--out`, результаты сохраняются рядом с исходным видео.

**Claude Code CLI:** используйте вместо этого `/plugin marketplace add perso-ai/perso-dubbing-plugin` и `/plugin install perso-dubbing@perso-ai`.

### Все остальные хосты

```bash
npx perso-dubbing
```

Определяет, какие хосты у вас есть, устанавливает оба навыка в каждый из них и выводит, куда именно попал каждый. Добавьте `--claude` / `--codex` / `--cursor` / `--antigravity` для одного хоста, `--all` для всех сразу или `--project` только для текущего проекта. Репозиторий уже клонирован? `node scripts/install.mjs` из корня репозитория делает то же самое, без обращения к сети.

> **🤖 Агенту:** если вам дали только URL этого репозитория:
>
> - **Claude Code (приложение для компьютера или CLI)** — **не** запускайте npx, даже если пользователь сказал «установи это». Покажите ему две команды `claude …` выше и попросите ввести каждую самостоятельно (это слэш-команды, которые вводит пользователь, и вы не можете их выполнить), а также путь с кликами: кнопка **+** → **Plugins** → **Add plugin**. Используйте npx только если после этого пользователь всё же хочет, чтобы установку выполнили вы.
> - **Codex · Cursor · Antigravity, или если пользователь поручил установку вам** — выполните `npx perso-dubbing` (или `node scripts/install.mjs`, если репозиторий уже клонирован).

### Первый запуск — ваш API-ключ

Откроется страница в браузере: войдите и нажмите одну кнопку — ключ будет выпущен и сохранён на этом компьютере в зашифрованном виде. Ничего копировать не нужно. Если браузер открыть не удаётся, вместо этого откроется файл ключа — вставьте туда ключ и сохраните, после чего он шифруется, а файл удаляется.

**Никогда не вставляйте свой API-ключ в чат.** → <a href="https://developers.perso.ai/api-keys" target="_blank" rel="noopener noreferrer">Получить API-ключ</a> · проверить в любой момент можно командой `npm run key:check`

## Использование

Просто скажите агенту, что вам нужно:

> «Продублируй это видео на английский — C:\videos\clip.mp4»
>
> «Продублируй все видео в этой папке на японский и испанский»
>
> «Продублируй эту ссылку с YouTube на английский, с синхронизацией губ»
>
> «Раздели голос и фоновую музыку в этом ролике»
>
> «Сделай мне английские субтитры SRT для этого видео»

Либо введите **`/dubbing`** / **`/srt`**, чтобы начать. Полный список опций CLI можно узнать у агента или командой `npm run dub -- --help`.

## Решение проблем

Остались вопросы? См. **[FAQ](FAQ.md)**.

| Симптом | Решение |
|---|---|
| `node` не найден | Установите LTS-версию с <a href="https://nodejs.org" target="_blank" rel="noopener noreferrer">nodejs.org</a> (или попросите агента: *«установи Node.js за меня»*), затем повторите попытку. |
| Приложение Claude для компьютера запрашивает Git (Windows) | При первом использовании вкладке Code нужен <a href="https://git-scm.com/downloads/win" target="_blank" rel="noopener noreferrer">Git для Windows</a>. Установите его и перезапустите приложение. |
| Команды `claude` или меню Plugins не реагируют | Вы находитесь в **облачной сессии** — плагинам нужна сессия **Local** (или SSH). |
| Ключ отклонён или отсутствует | Зарегистрируйте его заново: `node skills/dubbing/scripts/connect.mjs`. Проверить сохранённый ключ можно командой `npm run key:check`. |
| Ошибка, связанная с ffmpeg | Обычно ffmpeg устанавливается сам; если это не удалось, выполните `npm run doctor`. |
| Останавливается на середине (кончились кредиты, сбой, процесс завершён) | Прогресс сохраняется непрерывно. Выполните команду **`--resume "<state-file>"`**, указанную в уведомлении, — уже готовые части пропускаются и повторно не тарифицируются. |

## Конфиденциальность и телеметрия

`/dubbing` и `/srt` отправляют события использования, чтобы улучшать навыки — например, какое действие было выполнено, успешно ли оно завершилось, длительность медиафайла, версию приложения и ОС. Каждое событие содержит случайный идентификатор конкретной установки и номер вашего рабочего пространства. Ваш API-ключ и ваши медиафайлы никогда в них не включаются. Отказаться можно в любой момент с помощью `PERSO_NO_TELEMETRY`.
