`/dubbing`과 `/srt` 스킬에 대해 자주 묻는 질문입니다. 설치·사용법은 [README](README.md)를 참고하세요.

### 사용하려면 뭐가 필요한가요?

Node.js 18+ 와 Perso Dubbing API 키가 필요합니다. 스킬을 설치한 뒤 *"이 영상 더빙해줘"* 라고 말하면 됩니다. → [API 키 발급](https://developers.perso.ai/api-keys)

### API 키는 어떻게 등록하나요?

첫 실행 시 브라우저 페이지가 열립니다 — 로그인하고 한 번 클릭하면 키가 발급되어 이 PC에 암호화 저장됩니다. 복사할 것이 없습니다. 브라우저를 열 수 없는 환경이라면 대신 키 파일이 열립니다: 그 파일에 **API 키만** 붙여넣고 저장하세요(암호화되며 파일은 삭제됩니다). **키를 채팅에 붙여넣지 마세요.** 수동 확인: `npm run key:check`.

### 비용이 드나요?

스킬 코드 자체는 무료(MIT)지만, 더빙은 Perso API를 통해 실행되며 크레딧이 과금됩니다.

### 무엇을 입력할 수 있나요?

로컬 파일이나 로컬 폴더 또는 URL(유튜브, 틱톡, 구글 드라이브). 용량이 크거나 아주 긴 영상은 자동으로 분할·처리 후 다시 합쳐집니다.

### 여러 언어로 더빙하거나 여러 파일을 한 번에 처리할 수 있나요?

네. 한 명령에 여러 언어를 넣으면(`--target en,ja,zh`) 소스를 한 번만 업로드·분할해 언어별로 재사용합니다. 여러 파일·폴더·URL을 한 실행에 섞어도 됩니다.

### 결과는 어디에 저장되나요?

기본적으로 원본 영상 옆에 저장되며, `--out`으로 폴더를 지정할 수도 있습니다. 모든 실행은 Perso 포털(<https://perso.ai/en/workspace/vt>)의 프로젝트로도 남아, 재다운로드하거나 다른 형식을 받을 수 있습니다.

### 립싱크가 뭔가요?

입 모양을 더빙된 음성에 맞춰줍니다. 더빙 이후에 실행되고, 영상에만 적용되며, 시간이 상당히 더 걸리고 크레딧이 추가로 듭니다. `--lipsync`를 붙이세요.

### 음원 분리가 뭔가요?

소스를 음성 / 배경음 / 서브 배경음 트랙으로 분리합니다 — 더빙은 하지 않습니다. `--separate`를 붙이세요.

### 더빙 대신 자막(SRT)도 만들 수 있나요?

**`/srt`** 스킬을 이용해 영상/오디오/URL에서 Perso의 음성 인식으로 원본 언어 자막을 추출할 수 있습니다. 원하는 언어로 번역한 SRT도 필요하면, 원하는 언어와 함께 SRT 생성을 요청하세요.

### 중간에 멈췄어요 (크레딧 소진·크래시·셸 종료). 어떻게 하죠?

실행 내내 진행 상태가 상태 파일에 저장됩니다 (`/dubbing`은 `*.dubresume.json`, `/srt`는 `*.srtresume.json`). 안내에 표시된 `--resume "<state-file>"` 명령을 다시 실행하면 남은 부분만 마무리합니다 — 완료된 부분은 건너뛰고 다시 과금되지 않습니다.

### 크레딧이 떨어졌어요. 어떻게 충전하나요?

스킬이 Stripe 결제 링크를 생성해줍니다(플랜에 따라 구독·플랜 변경·크레딧 구매). 링크는 직접 열어 결제하시면 되고 — 에이전트가 대신 결제하지 않습니다. 충전 후에는 안내된 `--resume` 명령으로 이어서 진행하세요.

### 로컬에 파일을 저장하지 않고 더빙만 할 수 있나요?

네, 단일(분할되지 않은) 영상이라면 `--no-save`를 붙이세요. 결과는 Perso 워크스페이스에 남고 다운로드하지 않습니다. 분할된 영상은 병합본에 로컬 다운로드가 필요하므로 정상적으로 저장됩니다.

### `node`를 찾을 수 없다고 나와요.

스킬은 Node.js 18+ 가 필요합니다. `node -v`로 확인하고, 없으면 <https://nodejs.org>에서 LTS를 설치하거나 에이전트에게 설치를 요청한 뒤 다시 시도하세요.

### 스킬은 어떻게 업데이트하나요?

`npx perso-dubbing@latest`, 또는 Claude Code 플러그인에서 `/plugin update perso-dubbing`.

### 스킬이 어떤 데이터를 수집하나요?

사용 이벤트만 수집합니다 — 어떤 동작을 실행했는지, 성공 여부, 대략적인 개수, 앱 버전, OS — 설치별 랜덤 ID와 워크스페이스 번호로 태깅됩니다. API 키와 미디어 내용은 포함되지 않습니다.
