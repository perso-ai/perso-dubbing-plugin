关于 `/dubbing` 和 `/srt` 技能的常见问题。安装与使用方法请参见 [README](README.md)。

### 使用它需要什么条件？

Node.js 18+ 和一个 Perso Dubbing API 密钥。安装该技能后，直接说*“帮我配音这个视频”*即可。 → [获取 API 密钥](https://developers.perso.ai/api-keys)

### 如何注册我的 API 密钥？

首次运行时浏览器会打开一个页面——登录并点击一次，你的密钥便会签发并加密保存在本机。无需复制任何内容。如果无法打开浏览器，则会改为打开一个密钥文件：请在该文件中**只粘贴你的 API 密钥**并保存（文件会被加密后随即删除）。**切勿将密钥粘贴到聊天窗口中。**手动检查：`npm run key:check`。

### 使用它需要付费吗？

该技能的代码本身是免费的（MIT 许可证），但配音处理通过 Perso API 完成，会按积分计费。

### 可以输入哪些内容？

本地文件、本地文件夹，或一个 URL（YouTube、TikTok、Google Drive）。体积过大或时长过长的视频会被自动拆分、处理并重新合并。

### 能否一次配多种语言，或一次处理多个文件？

可以。在一条命令中填入多种语言（`--target en,ja,zh`）——源文件只会上传和拆分一次，然后在各语言之间复用。你也可以在一次运行中混合使用多个文件、文件夹和 URL。

### 结果保存在哪里？

默认保存在源视频旁边，或保存到你通过 `--out` 指定的文件夹中。每次运行同时也会在你的 Perso 门户网站（<https://portal.perso.ai>）中生成一个项目，你可以在那里重新下载或获取其他格式。

### 什么是唇形同步（lip-sync）？

它会让口型与配音后的音频相匹配。该处理在配音完成之后进行，仅适用于视频，耗时明显更长，且会额外消耗积分。添加 `--lipsync` 即可启用。

### 什么是音频分离？

它会将源文件拆分为人声 / 背景 / 次背景音轨，不涉及配音处理。添加 `--separate` 即可启用。

### 能不能只生成字幕（SRT），而不是配音？

**`/srt`** 技能会通过 Perso 的语音转文字功能，从视频/音频/URL 中提取原始语言的字幕。如果你还需要翻译，请在提出 SRT 需求时一并说明你想要的语言。

### 运行到一半停止了（积分不足、崩溃，或 shell 被终止）。该怎么办？

运行过程中，进度会持续保存到状态文件中（`/dubbing` 对应 `*.dubresume.json`，`/srt` 对应 `*.srtresume.json`）。重新运行提示中给出的 `--resume "<state-file>"` 命令，即可仅完成剩余部分——已完成的部分会被跳过，且不会重复计费。

### 我的积分用完了，该怎么充值？

该技能可以生成一个 Stripe 支付链接（根据你的套餐，可用于订阅、变更套餐或购买积分）。你需要自行打开链接并完成付款——智能体不会代你付款。充值完成后，使用提示中给出的 `--resume` 命令继续任务。

### 可以在不保存本地文件的情况下进行配音吗？

可以，仅限单个（未拆分）的视频：添加 `--no-save` 即可。结果会保留在你的 Perso 工作区中，不会被下载。而拆分处理过的视频仍会正常保存，因为合并后的文件需要本地下载。

### 提示找不到 `node`，该怎么办？

该技能需要 Node.js 18+。可通过 `node -v` 检查；从 <https://nodejs.org> 安装 LTS 版本，或者直接请智能体帮你安装后重试。

### 如何更新该技能？

运行 `npx perso-dubbing@latest`，或在 Claude Code 插件中执行：`/plugin update perso-dubbing`。

### 该技能会收集哪些数据？

仅收集使用事件——包括执行了哪个操作、是否成功、粗粒度的计数、应用版本以及操作系统——并通过一个随机生成的每次安装唯一 ID 和你的工作区编号进行标记。你的 API 密钥和媒体内容绝不会包含在内。
